# Source-Backed Research

Use this reference when a generated skill depends on current docs, standards, library APIs, academic claims, or public distribution behavior.

## Source Priority

1. Official product docs and specs.
2. Official example repositories.
3. Maintainer docs for libraries or tools.
4. Well-cited research papers or engineering writeups.
5. Practitioner feedback as pattern signal, not sole authority.

## Source Record

For each material source, record:

- title
- URL or local path
- date checked
- source type
- confidence
- what it supports
- outdated-risk

## Fact vs Interpretation

Separate:

- fact: what the source states or demonstrates
- interpretation: what it means for the requested skill
- impact: what changes in the generated package

## Stop Rule

Stop researching when the evidence is enough to make a defensible design and implementation plan. Do not loop forever looking for marginal examples.
